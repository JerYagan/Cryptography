How Fractal Bloom Cipher Works

A simple walkthrough of how your message turns into a fractal image — and back.

1. Convert Text → Bytes

Your message is turned into raw bytes. This is the basic data computers understand.

2. Create AES Key From Password

Your password is stretched using PBKDF2 (slow hashing) to generate a strong AES-256 key.

3. Encrypt the Message

AES-GCM locks your bytes into unreadable ciphertext. Without the password, it’s impossible to recover.

4. Generate Fractal From Password

The password also produces a numeric seed. This seed shapes the Julia fractal’s colors and movement.

5. Render the Fractal

WebGL draws a smooth animated fractal using the seed. This part is purely visual — no data hidden yet.

6. Hide Ciphertext in Pixels

The encrypted bytes are embedded inside the image by toggling tiny bits in pixel colors (LSB steganography).

7. Save as PNG

The result looks like a normal fractal image, but it secretly contains your encrypted data.

8. Decode + Decrypt

When decoding, the app extracts the hidden bytes and decrypts them using the same password.

In short: your message is encrypted with AES, then hidden inside the fractal’s pixels. Only the correct password can unlock it.
