1. Text ‚Üí Bytes

Your message is converted into raw bytes using UTF-8. This is the basic form cryptography operates on.

2. Password ‚Üí Key Stream

Your password is turned into bytes and repeated to match the message length. This acts as a simple key stream.

3. XOR Encryption

Each message byte is XOR‚Äôed with a password byte. The result is unreadable without the same password.

4. Password ‚Üí Seed

The password is also hashed into a numeric seed, which controls the Julia fractal‚Äôs structure and motion.

5. Render Fractal

WebGL draws a smooth neon Julia fractal on the canvas using the password-derived seed. Still clean ‚Äî no data hidden yet.

6. Hide Data in Pixels

The encrypted bytes are embedded into the least significant bits (LSBs) of pixel colors. The image still looks identical to the eye.

7. Save PNG

The fractal PNG now carries XOR-encrypted bytes invisibly in its pixel data. You can share it like normal art.

8. Decode + Decrypt

The app reads the LSBs, reconstructs the encrypted bytes, then XOR-decrypts them with the password to recover your original text.

In short: your message is XOR-encrypted, then hidden inside a fractal image. Only the correct password can reverse the XOR and reveal the text.

Advanced Technical Breakdown
üîê XOR Cipher

The cipher uses a simple XOR operation between message bytes and password bytes:

  • Password is encoded as bytes and repeated to match the message length.
  • Each plaintext byte is XOR‚Äôed with the corresponding password byte.
  • Decrypting runs the same XOR again with the same password.
üß¨ Password-Derived Seed

The password is hashed into a float seed value in [0,1], which controls the Julia fractal:

  • Base complex parameter c of the fractal
  • Subtle motion, zoom, and rotation over time
  • Color oscillations and palette blending
üé® Fractal Rendering

A WebGL fragment shader iterates the Julia set formula for each pixel and colors it using a Turbo colormap plus neon domain coloring.

üñº LSB Steganography

Encrypted bytes are hidden inside the RGB channels of pixels by modifying only the least significant bit: 

pixel.r = (pixel.r & 0xFE) | bit;

Because only the last bit is changed, the visual color difference is imperceptible to human eyes.

üì¶ Pixel Data Layout

The first 32 bits embedded in the pixels store the length of the encrypted payload. The remaining bits store the XOR ciphertext bytes.

üì• Decoding Flow

On decode, the app:

  • Reads pixel LSBs and reconstructs the embedded bytes.
  • Uses the user-provided password to XOR-decrypt the bytes.
  • Converts the result back to UTF-8 text.
