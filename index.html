1. Text ‚Üí Bytes

Your message is turned into raw bytes. This is the basic data used in cryptography.

2. Password ‚Üí AES Key

Your password is stretched with PBKDF2 (slow hashing) to generate a strong AES-256 key.

3. AES-GCM Encryption

The bytes are encrypted into ciphertext. Without the exact password, decryption fails.

4. Password ‚Üí Seed

The password also generates a fractal seed, affecting the fractal‚Äôs shape & motion.

5. Render Fractal

WebGL draws a smooth neon Julia fractal. Still clean ‚Äî no message hidden yet.

6. Hide Data in Pixels

Hidden using tiny changes in pixel bits (LSB steganography). Looks identical to the eye.

7. Save PNG

The fractal PNG now carries encrypted bytes invisibly inside its pixel data.

8. Decode + Decrypt

Extracts hidden bytes ‚Üí decrypts with AES-GCM ‚Üí returns your original message.

In short: your message is encrypted, then hidden inside a fractal image. Only the correct password can unlock it.
